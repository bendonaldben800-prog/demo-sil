# Device-aware Download Portal

Static website that detects user platform and presents the most relevant app download link.

## Features

- platform detection for Windows and macOS
- smart primary CTA based on detected platform
- manual platform picker fallback
- manifest-driven download links (`manifest.json`)
- checksum list for verification

## Local run

Use any static server:

```bash
cd download_portal
python3 -m http.server 8080
```

Then open `http://localhost:8080`.

## Release update flow

1. Build and publish new app artifacts.
2. Update URLs and checksums in `manifest.json`.
3. Deploy this folder to your static host.

## Windows artifacts source

Windows download files are generated by:

- .github/workflows/windows-artifacts.yml

It creates:

- key-monitor-win-x64.zip
- key-monitor-win-arm64.zip
- key-monitor-win-x64-setup.exe
- key-monitor-win-arm64-setup.exe
- matching .sha256.txt files
